# 卡牌单词分类 Demo — 设计文档

## 1. 技术选型

| 选型 | 理由 |
|------|------|
| **Vite + React + TypeScript** | 上手快、热更新利于调交互；类型约束适合表达「牌 / 柱 / 关卡」模型，避免状态散落。 |
| **DOM + CSS 动效** | 卡牌量小、以布局与拖拽为主，不必上 Canvas/WebGL；CSS 动画易做翻入、落栈、失败震动，且易做 `prefers-reduced-motion`。 |
| **手写 Pointer 拖拽** | 本题核心是「拖放 + 吸附判定」，未引入 `@dnd-kit` 等库，减少依赖与样板代码；用 `setPointerCapture` 保证触屏跟随。 |

## 2. 核心玩法实现思路

### 2.1 状态机（简化）

- **playing**：可抽牌、可拖动落子。
- **won**：牌堆空、手牌空、且所有单词卡已进入柱。
- **lost**：步数 ≤ 0 且未达成胜利。

### 2.2 数据结构

- **deck**：`PlayingCardModel[]`，索引 `0` 表示**下一张将被抽出**的牌顶。
- **hand**：当前必须处理的一张牌；与原作一致，**先处理再抽**，避免多张悬而未决。
- **pillars**：三根柱，每柱 `lockedCategoryId`（是否已放分类卡）与 `placedWords[]`（该柱已接纳的单词）。

### 2.3 规则校验（`gameModel.ts`）

- **分类卡**：仅可落在 `lockedCategoryId === null` 的柱。
- **单词卡**：柱必须先有分类，且 `word.categoryId === pillar.lockedCategoryId`。
- **成功落子**：清空 `hand`，`movesLeft -= 1`，再判断是否 win/lose。
- **失败落子**：不扣步、不消费手牌；`rejectPulse++` 驱动 UI 震动。

单关数据与牌序见 `src/game/level.ts`：牌序故意按「分类 → 该组单词 → 下一分类…」排列，保证可解、便于第一次体验。

## 3. 交互与手感取舍

| 设计点 | 取舍 |
|--------|------|
| **吸附** | 以柱节点 `getBoundingClientRect()` 为命中盒，外扩 `pad` 像素，减少「边缘松手失败」的挫败感。 |
| **多柱重叠** | 若同时命中多个柱（窄屏叠放时少见），取**面积较小**的 rect，优先落在更「像目标」的那一列。 |
| **误放不扣步** | 与「步数资源」形成对比：惩罚是时间与操作次数感，而非直接扣步，降低误触成本。 |
| **抽牌门槛** | 仅当 `hand === null` 可抽，强制单卡决策，节奏接近接龙式「一次处理一张」。 |

## 4. 动效（Juice）方案

| 节点 | 实现 |
|------|------|
| **亮牌** | 手牌 `handCard--deal` + `flip-in`（内层 `rotateY` + 轻微模糊渐隐）。 |
| **拖动** | 父层 `translate` + 随横向位移微微 `rotate`，`filter: drop-shadow` 抬升感。 |
| **误放** | `rejectPulse` 变更触发 `handCard__inner--shake`（横向位移 + 微旋转）。 |
| **单词入柱** | `miniCard` 使用 `stack-in`（自下方缩放入场）。 |
| **胜负** | 顶栏 `toast` 使用 `toast-in`；终局 `board__veil` 轻遮罩防止误操作。 |
| **无障碍** | `@media (prefers-reduced-motion: reduce)` 下将动画时长压到可忽略。 |

若继续迭代，可加重：柱在 drag 经过时高亮、成功落子粒子、deck 顶牌短促抬起等。

## 5. Gameplay 理解与可优化方向

### 5.1 核心乐趣（个人理解）

1. **双重约束下的轻量规划**：牌序部分可见（逐张翻开），柱位有限，分类卡放哪根柱会改变后续单词的「动线」，是短视与一步棋的博弈。
2. **语义分类的即时正反馈**：单词落到正确柱时，类别与词义在脑中「咔哒」对齐，类似整理收纳的爽感。
3. **接龙式节奏**：一次只处理一张牌，认知负荷低，适合碎片时间；步数上限提供温和紧张感。

### 5.2 可优化方向（Demo 外）

- **信息节奏**：适当透露「剩余牌堆中各类数量」或下一张轮廓，增加规划深度。
- **容错与教学**：首次误放给出柱高亮提示；分类卡暂存区（有限）降低死锁。
- **关卡生成**：在可解性约束下洗牌 / 生成牌序，并调节步数上限与柱数。
- **长期目标**：连击、星级、主题皮肤等，服务于留存。
